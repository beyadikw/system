# بوابة طلبات رعاية الفعاليات — مشروع خذ بيدي

نظام ويب متكامل لاستقبال ومراجعة طلبات رعاية الفعاليات وإصدار التقارير الختامية، تحت مظلّة مشروع **خذ بيدي** (ثلث المرحوم عبدالله عبداللطيف العثمان).

يتكوّن من واجهة عربية (RTL) بالكامل، وخادم Express، وقاعدة بيانات MySQL، مع رفع ملفات وإشعارات بريدية وتحليلات بيانية.

---

## ✨ المزايا

- **بوابة فريق المشروع** (لوحة متابعة + إدارة الطلبات + التقارير) — `index.html`
- **نموذج تقديم طلب عام** متعدّد الخطوات يبدأ بالموافقة على الشروط والأحكام
- **رابط مخصّص للجهة المنفّذة** لرفع تقرير الفعالية وصوره (`report.html?token=...`)
- **لوحة متابعة بالرسوم البيانية**: الحالات، التوزيع الشهري، القاعات، الفئات المستهدفة
- **رفع الملفات**: طلب موقّع + سيرة ذاتية + حتى ٦ صور للتقرير
- **إشعارات بريدية**: طلب جديد، تغيير الحالة، رابط التقرير، استلام التقرير
- **تصدير التقرير** PDF و Word مع شعار المشروع
- **مصادقة JWT** وصلاحيات (منسّق / مدير / مطّلع)

## 🚀 التشغيل

### المتطلّبات
- Node.js 18+
- MySQL 8+

### الخطوات

```bash
cd app
cp .env.example .env          # ثم اضبط بيانات قاعدة البيانات و JWT_SECRET
npm install

# تهيئة قاعدة البيانات + البيانات الأولية
npm run db:seed

# التشغيل
npm run dev                   # تطوير (إعادة تحميل تلقائي)
# أو
npm start
```

ثم افتح: **http://localhost:4000/login.html**

> **بيانات الدخول التجريبية:** `coordinator@beyadik.kw` · `Beyadik@2026`

### وضع البريد
إن تركت `SMTP_HOST` فارغاً، تُطبع الرسائل في الـ console وتُسجّل في جدول `email_log` — مناسب للتطوير دون خادم بريد.

## 🔌 واجهة API

كل المسارات تحت `/api`. المسارات المحميّة تتطلّب ترويسة `Authorization: Bearer <token>`.

| الطريقة | المسار | الوصف | الحماية |
|---|---|---|---|
| POST | `/api/auth/login` | تسجيل الدخول | عام |
| GET | `/api/auth/me` | المستخدم الحالي | محميّ |
| GET | `/api/meta` | القاعات والفئات | عام |
| POST | `/api/requests` | تقديم طلب (multipart) | عام |
| GET | `/api/requests?status=` | قائمة الطلبات | محميّ |
| GET | `/api/requests/:id` | تفاصيل طلب | محميّ |
| PATCH | `/api/requests/:id/status` | تحديث الحالة | محميّ |
| PUT | `/api/requests/:id` | تعديل حقول الطلب | محميّ |
| DELETE | `/api/requests/:id` | حذف الطلب | محميّ |
| POST | `/api/requests/:id/share` | توليد/إرسال رابط التقرير | محميّ |
| GET | `/api/reports` | الطلبات المعتمدة + حالة التقرير | محميّ |
| POST | `/api/reports/:requestId` | حفظ تقرير (صور multipart) | محميّ |
| GET | `/api/share/:token` | بيانات الفعالية للجهة المنفّذة | عام |
| POST | `/api/share/:token` | رفع التقرير من الجهة المنفّذة | عام |
| GET | `/api/analytics/summary` | مؤشرات لوحة المتابعة | محميّ |
| GET | `/api/analytics/timeseries` | الطلبات شهرياً | محميّ |
| GET | `/api/analytics/by-hall` | التوزيع حسب القاعة | محميّ |
| GET | `/api/analytics/by-category` | التوزيع حسب الفئة | محميّ |

## 🔌 رابط النظام بعد النشر

- **بوابة المنسّق (تسجيل دخول):** `https://رابطك.onrender.com/login.html`
- **صفحة تقديم طلب للجهات (عامة، بلا حساب):** `https://رابطك.onrender.com/submit.html`
- **رابط رفع تقرير للجهة المنفّذة:** يُولّد من داخل البوابة لكل فعالية.

> البوابة المستضافة مضبوطة على `APP_MODE = 'live'` فتعمل على البيانات الحقيقية تلقائياً.

## 🔗 ربط الواجهة بالخادم (DEMO ↔ LIVE)

تعمل بوابة `index.html` افتراضياً في **وضع تجريبي** (`window.APP_MODE = 'demo'`) ببيانات داخلية، لتُعرض دون خادم.

**الربط الحيّ مكتمل وجاهز:** غيّر `window.APP_MODE` إلى `'live'` في `public/index.html` فقط — وستعمل كل الشاشات (لوحة المتابعة، الطلبات، التقارير) على بيانات قاعدة البيانات الحقيقية، وتُحفظ جميع الإجراءات (اعتماد/رفض/تعديل/حذف/تقرير/رفع صور) في الخادم. طبقة الربط في `public/js/data.js` + `public/js/api.js`.

> صفحتا `login.html` و `report.html` تعملان في الوضع الحيّ مباشرةً.

📄 **للنشر الكامل خطوة بخطوة (HTTPS، النطاق، PM2، التأمين): راجع `DEPLOYMENT.md`.**

## 🔒 ملاحظات أمنية للإنتاج
- غيّر `JWT_SECRET` إلى قيمة عشوائية طويلة، وكلمة مرور المنسّق بعد أول دخول.

- خزّن المرفقات خارج المجلّد العام أو على تخزين سحابي في الإنتاج.
